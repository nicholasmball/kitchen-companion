Clickable Timeline Steps

UX Design Document — Building on Requirements & Acceptance Criteria (Step 1)

Design Rationale
Problem: Timeline events ("Start prepping potatoes") are display-only. Cooks must navigate away to the recipe page to check ingredients or instructions.

Solution: Make event cards clickable (R1). Tap opens a bottom Sheet overlay (R2) with the side-by-side ingredient/instruction layout, consistent with the cooking mode we just built. The sheet slides up from the bottom — the user stays on the timeline and can dismiss to return.

Key decisions:
• Sheet, not modal: A bottom sheet preserves timeline context (user can see the top of the timeline behind the overlay). Modals feel more disruptive on mobile.
• Chevron hint on clickable events: A small › chevron on the right side signals interactivity without cluttering the card. Only shown on events with recipe data.
• Hover state: Clickable cards get a subtle border colour change + shadow + slight translate on hover (R1).
• Event type badge in sheet header: e.g. "Prep" or "Cook" — gives context for which phase this is (R2).
• Highlighted instruction: The relevant step is auto-scrolled to and highlighted with a left border accent (R3), matching the active step pattern from cooking mode.
• Ingredient checkboxes: Green checkboxes, consistent with cooking mode (R5). Session-only state, reset when sheet closes.

Timeline: Clickable Events
Clickable cards: Events where the meal item has instructions get cursor: pointer, hover border/shadow, and a chevron hint. Events without recipe data (e.g. custom items with no instructions) remain non-interactive.
"Serve!" event: Always non-clickable (it's a status event, not a recipe step).

Roast Potatoes

Cook
Ingredients
✓
1 kg Maris Piper potatoes
3 tbsp goose fat
1 tsp sea salt flakes
1/2 tsp garlic granules
2 sprigs fresh rosemary
Instructions
1 Peel and quarter the potatoes. Place in a large pan of salted water.
2 Bring to the boil and parboil for 8 minutes. Drain and shake in the colander to roughen edges.
3 Heat goose fat in a roasting tin at 220°C for 5 minutes until smoking hot.
4 Carefully add potatoes to the hot fat, turning to coat. Sprinkle with salt and garlic granules.
5 Roast for 45-50 minutes, turning halfway through. Add rosemary sprigs for the last 15 minutes.
6 Remove when golden and crispy. Drain on kitchen paper and serve immediately.

Component Architecture
Modified: components/planner/timeline-view.tsx — add click handler + hover styles + chevron to event cards
New: components/planner/timeline-recipe-sheet.tsx — the Sheet overlay component

Props for TimelineRecipeSheet:
open: boolean
onOpenChange: (open: boolean) => void
mealItemName: string
eventType: TimelineEventType
ingredients: Ingredient[] | null
instructions: string | null

Data flow: Extend TimelineEvent type with ingredients?: Ingredient[] and instructions?: string. The calculateTimeline() function already receives MealItem[] which has both fields — just pass them through to the events.

Step highlighting heuristic:
• prep_start → highlight step index 0
• cook_start → find first step containing "oven", "hob", "heat", "fry", "bake", "roast", "grill"; fallback to middle step
• cook_end / rest_start → highlight last step
• serve → not clickable
Accessibility
• Clickable event cards: role="button", tabIndex={0}, aria-label="View recipe details for {description}"
• Keyboard: Enter/Space opens the sheet
• Sheet: uses shadcn Sheet component which handles focus trap, Escape to close, aria-describedby
• Ingredient checkboxes: role="checkbox", aria-checked, aria-label (same as cooking mode)
• Non-clickable events: no role/tabIndex — they remain plain divs
• Highlighted step: aria-current="step" for screen readers
