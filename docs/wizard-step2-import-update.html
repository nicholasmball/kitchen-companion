Wizard Step 2 Update

Two changes: scrollable recipe grid + import options (URL / Image / Photo)

What Changed
1. Scrollable recipe grid: The recipe grid is now inside a scrollable container (max-height: 400px; overflow-y: auto) so ALL recipes show, not just the first 6. A subtle scroll hint "Scroll to see more" appears when there are more than 6 recipes.

2. Import section: A new "Import a new recipe" panel sits between the recipe grid and the custom item form. Three import options:
• From URL — paste a recipe URL, uses existing /api/parse-recipe-url endpoint
• From Image — upload a photo of a recipe, uses existing /api/parse-recipe-image endpoint
• Take Photo — capture with device camera (same image endpoint, but triggers camera input)

All three use the existing RecipeImporter component which already handles both URL and image parsing. The imported recipe gets saved to the user's recipe collection AND added to the plan simultaneously.

Layout order (top to bottom): Search bar → Scrollable recipe grid → Import section → Custom item form → Your Dishes panel

Implementation Notes
Scrollable grid: Wrap the recipe grid in a div with max-h-[400px] md:max-h-[400px] max-h-[50vh] overflow-y-auto. Remove any limit on the number of recipes rendered.

Import flow: Reuse the existing RecipeImporter component. When the user imports a recipe:
1. Save it to the user's recipe collection (via useRecipes().createRecipe())
2. Automatically add it to the wizard's items array as a WizardItem
3. Close the importer dialog

Import buttons trigger:
• "From URL" → opens RecipeImporter with URL tab active
• "From Image" → opens RecipeImporter with Image tab active
• "Take Photo" → opens RecipeImporter with Image tab, but triggers capture="environment" on the file input to open camera directly

Scroll hint: Only shown when recipes.length > 6. Shows count: "Scroll to see all {n} recipes".
